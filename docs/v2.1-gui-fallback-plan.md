# v2.1: GUI Fallback for Unsupported Protocol Versions

## Problem

Protocolize doesn't support all Minecraft protocol versions immediately. When a player connects with a newer client (e.g., 1.21.5 with protocol 774), the GUI fails with:

```
IllegalStateException: Cannot send dev.simplix.protocolize.data.packets.OpenWindow to players with protocol version 774
```

## Solution

Add a fallback path that skips the GUI and uses text-based confirmation when the GUI cannot be opened.

## Flow

### Current Flow (GUI works)
```
/apply "Name" "Inviter" → GUI opens → Player selects servers → Submit
```

### New Flow (GUI fails)
```
/apply "Name" "Inviter" → GUI fails → Text confirmation shown → Auto-submit with default servers
```

## Implementation

### 1. Modify `GuiManager.openApplicationGui()`

Wrap the GUI opening in try-catch. If it fails due to protocol version, return a result indicating fallback is needed.

```java
public boolean openApplicationGui(Player player, ApplicationData data, List<String> availableServers) {
    // ... existing code ...

    try {
        gui.open();
        return true; // GUI opened successfully
    } catch (IllegalStateException e) {
        if (e.getMessage() != null && e.getMessage().contains("protocol version")) {
            openGuis.remove(uuid);
            return false; // Signal fallback needed
        }
        throw e; // Re-throw other errors
    }
}
```

### 2. Modify `ApplyCommand.handleApply()`

Check the return value and handle fallback:

```java
boolean guiOpened = guiManager.openApplicationGui(player, data, availableServers);

if (!guiOpened) {
    // GUI failed - use text fallback
    handleTextFallback(player, data, availableServers);
}
```

### 3. Add `ApplyCommand.handleTextFallback()`

New method that:
1. Shows application summary in chat
2. Lists default servers that will be requested
3. Submits immediately (or asks for confirmation via clickable text)

```java
private void handleTextFallback(Player player, ApplicationData data, List<String> servers) {
    // Get default servers from config
    List<String> defaultServers = config.getDefaultServers();
    if (defaultServers.isEmpty()) {
        defaultServers = servers; // Use all available if no defaults
    }

    // Show summary
    player.sendMessage(Component.empty());
    sendMessage(player, "&eGUI unavailable - using text mode");
    player.sendMessage(Component.empty());
    player.sendMessage(colorize("&7Name: &f" + data.realName()));
    player.sendMessage(colorize("&7Inviter: &f" + data.inviter()));
    if (data.discord() != null) {
        player.sendMessage(colorize("&7Discord: &f" + data.discord()));
    }
    player.sendMessage(colorize("&7Servers: &f" + String.join(", ", defaultServers)));
    player.sendMessage(Component.empty());

    // Submit directly
    submitApplicationDirect(player, data, defaultServers);
}
```

### 4. Add `ApplyCommand.submitApplicationDirect()`

Move submission logic from GuiManager to a shared location, or duplicate minimally:

```java
private void submitApplicationDirect(Player player, ApplicationData data, List<String> servers) {
    try {
        // Check for pending application (already done in handleApply, but double-check)
        Optional<Application> pending = applicationRepository.findPendingByUuid(player.getUniqueId());
        if (pending.isPresent()) {
            sendMessage(player, "&eYou already have a pending application.");
            return;
        }

        // Create and save application
        String serversNote = "Requested servers: " + String.join(", ", servers);
        String fullNotes = data.notes() != null
            ? data.notes() + " | " + serversNote
            : serversNote;

        Application application = Application.createPending(
            player.getUniqueId(),
            data.realName(),
            data.discord(),
            data.inviter(),
            fullNotes
        );

        application = applicationRepository.insert(application);

        // Send confirmation
        player.sendMessage(Component.empty());
        sendMessage(player, "&aApplication submitted!");
        player.sendMessage(colorize("&7Your request is now pending review."));
        player.sendMessage(colorize("&7Use &f/apply status &7to check anytime."));

        logger.info("Player {} submitted application #{} via text fallback for servers: {}",
            player.getUsername(), application.id(), servers);

        // Notify Discord (need access to notifier)
        // This requires passing DiscordNotifier to ApplyCommand or using an event

    } catch (SQLException e) {
        logger.error("Failed to submit application for {}", player.getUsername(), e);
        sendMessage(player, "&cFailed to submit. Please try again.");
    }
}
```
