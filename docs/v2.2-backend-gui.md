# v2.2: Backend (Paper) GUI — replacing the Protocolize proxy GUI

## Why

The application GUI used to be injected from **Velocity via Protocolize**, sending the
`OpenWindow` packet directly to the client. That packet must match the client's exact
protocol version, so on any client/server protocol Protocolize didn't support yet it failed
with `Cannot send OpenWindow to players with protocol version <n>` and the plugin fell back
to chat — permanently, for everyone on newer clients (and on Paper 26.x).

This version moves the player-facing GUI to a **backend Paper plugin** on the lobby. A native
Bukkit inventory is protocol-agnostic (the server renders it), so there's no protocol treadmill,
and Geyser translates backend inventories to Bedrock forms reliably. No Protocolize.

## Architecture

```
FancyNPC (lobby) --click--> /gkmenu or /apply  (GatekeeperPaper, backend lobby)
        |                         |
        |                         +-- GET  /api/players/{uuid}/application-context
        |                         +-- POST /api/applications        (X-Shared-Secret)
        |                                   |
        v                                   v
   Bukkit inventory GUI            Velocity Gatekeeper (data, Discord, admin API)
   (Java) / Geyser form (Bedrock)
```

- **Velocity** stays the source of truth: SQLite data, entitlement gating on
  `ServerPreConnect`, the Discord bot integration, and `/access` admin commands.
- **GatekeeperPaper** owns the player UX: `/apply`, `/apply status`, `/gkmenu`.

## New Velocity API endpoints

Both require the `X-Shared-Secret` header (same secret as the Discord bot).

- `GET /api/players/{uuid}/application-context`
  → `{ restrictedServers, currentAccess, availableServers, defaultServers, hasPending, latest }`
- `POST /api/applications`
  body: `{ uuid, username, platform, realName, inviter, discord?, notes?, servers[] }`
  → `{ success, applicationId }` or `409 { success:false, error }` if already pending.

`/apply` is no longer registered on the proxy (see GatekeeperPlugin) so it reaches the backend.

## Deploying

1. **Velocity**: replace the proxy jar with `velocity/build/libs/access-manager-velocity-2.2.0-SNAPSHOT.jar`
   and restart the proxy. (Adds the two endpoints; no config change required.)

2. **Lobby Paper server**: drop `paper/build/libs/gatekeeper-paper-1.0.0-SNAPSHOT.jar` into the
   lobby's `plugins/` and start once to generate `plugins/GatekeeperPaper/config.yml`.

3. **Configure** `plugins/GatekeeperPaper/config.yml`:
   - `api.base_url`: how the lobby container reaches the Velocity admin API. On the shared
     Docker network this is the Velocity container/service name + port, e.g.
     `http://games-crafty-tvc67g-velocity-1:8080`.
   - `api.shared_secret`: **must match** `discord.shared_secret` in the Velocity config.
   - `worlds`: one entry per destination. `velocity_name` is the Velocity server name used to
     connect; `logical_id` must match the Velocity logical IDs (restricted/mapping).

4. **Velocity** must have the BungeeCord plugin-message channel enabled (default `true` in
   `velocity.toml`) so `/gkmenu` can route players to servers.

## Wiring the NPCs (FancyNpcs)

- **World portal NPC**: run-command-as-player → `gkmenu`. Opens the world menu; clicking a world
  you can access connects you, a locked world tells you to apply.
- **Apply NPC** (optional): since the GUI needs your name/inviter, point players to
  `/apply "YourName" "WhoInvitedYou"`, which opens the server-selection GUI. (A future version
  can collect those fields with a Bedrock form / anvil so the NPC can open it directly.)

## Deferred (future / optional)

- **Bedrock-native Cumulus forms** via Floodgate (nicer than Geyser-translated inventories),
  and **anvil text input** on Java so the whole flow is click-only without the `/apply` args.
- Remove the now-unused Velocity-side Protocolize GUI code (`gui/ApplicationGui`,
  `gui/GuiManager`, `gui/ItemBuilder`) and the `protocolize` dependency.
- Share the duplicated `colorize()` helpers via Adventure's `LegacyComponentSerializer`.
